# Практика: Логистическая регрессия с нуля

## Описание проекта
В данном проекте реализован алгоритм логистической регрессии для решения задачи бинарной классификации. Основная цель работы — глубокое понимание внутренних механизмов алгоритма путем его написания «с нуля» на чистом Python (с использованием библиотеки NumPy) и последующего сравнения с оптимизированной реализацией из библиотеки `scikit-learn`.

## Набор данных
В работе используется датасет **Breast Cancer Wisconsin**, доступный в библиотеке `scikit-learn`. 
* **Задача**: Бинарная классификация (доброкачественная или злокачественная опухоль).
* **Признаки**: Числовые метрики параметров клеточных ядер.

## Стек технологий
* **Python 3**
* **NumPy** & **Pandas** — векторные вычисления и работа с табличными данными.
* **Scikit-learn** — предобработка данных, расчет метрик и эталонная модель (`LogisticRegression`).
* **Matplotlib** — визуализация метрик и графиков.

## Что было реализовано
1. **Подготовка данных**: Разделение на обучающую и тестовую выборки с сохранением баланса классов (stratify), стандартизация признаков (`StandardScaler`).
2. **Математический аппарат**:
   * Сигмоидная функция активации (Sigmoid).
   * Функция потерь Log-Loss (с защитой от логарифма нуля).
   * Вычисление градиентов по весам и смещению.
3. **Модель классификатора**: Класс `MyLogisticRegressionGD`, обучающийся методом градиентного спуска. Реализованы методы `fit`, `predict_proba` и `predict`.
4. **Сравнение моделей**: Оценка качества (Accuracy, Precision, Recall, F1, ROC-AUC) собственной модели и `sklearn.linear_model.LogisticRegression`.
5. **Анализ порога**: Исследование влияния порога классификации (0.3, 0.5, 0.7) на метрики.

## Основные выводы
* Собственная реализация показывает метрики качества, сопоставимые с эталонной моделью из `scikit-learn` (разница в долях процентов). 
* Реализация из `sklearn` работает быстрее и стабильнее благодаря использованию продвинутых оптимизаторов (например, L-BFGS) и встроенной L2-регуляризации по умолчанию.
* Смещение порога классификации позволяет балансировать между Precision и Recall в зависимости от бизнес-требований задачи (например, минимизация ложноотрицательных результатов в медицине).
